## Sobre mim

Construo estudos de caso de QA com foco em três frentes:

- planejamento baseado em risco
- execução documentada com evidências
- automação web e API com leitura simples e reprodução rápida

Este portfolio documenta como penso qualidade de ponta a ponta: o que priorizar, como validar, como comunicar risco e como transformar resultado técnico em material útil para o time.

## Case Study 1

**Practice Software Testing em 3 camadas**

Um estudo de caso construído sobre um único produto público para documentar como QA manual, testes de API e testes E2E se complementam.

**English summary:** One public product tested across manual QA, API automation, and web E2E automation to show layered quality coverage.

### Repositórios do Case Study 1

| Repositório | Papel no estudo de caso | Resultado |
|---|---|---|
| [toolshop-quality-portfolio](https://github.com/Wesleysc94/toolshop-quality-portfolio) | Hub do estudo de caso com visão geral, mapa da qualidade e ordem de revisão | Ponto de entrada do case |
| [toolshop-manual-quality](https://github.com/Wesleysc94/toolshop-manual-quality) | Planejamento, análise de risco, smoke manual, bug reports e recomendação de release | **2 defeitos** · **No Go** |

## Case Study 2

**SwagLabs + JSONPlaceholder**

Um estudo de caso com dois produtos públicos de comportamento diferente: um front-end com bugs intencionais e uma API fake usada para praticar CRUD, criação e cenários negativos.

**English summary:** A two-product QA case study combining E2E automation on a bug-driven web app and API automation on a public fake API.

### Repositório principal
